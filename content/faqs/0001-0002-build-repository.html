--- group: 0001-Installation title: 0002-How do I build from the repository? kind: faq ---

First follow the instructions on installing from the sources.

You don't need to do a build to use the programs, so if you are not packaging a release you can just copy and/or use sbin/firehol.in and sbin/fireqos.in directly.

To build the documentation and/or a tar-file, you will need xsltproc, dblatex, DocBook XML, DocBook XSL Stylesheets, GNU autoconf and GNU automake. You can create the Makefiles, enable building the manual and make the documentation with these commands: 

./autogen.sh
./configure --enable-maintainer-mode
make

Note that versions checked out from git will use internal versions e.g.: 

$ ./sbin/fireqos.in
FireQOS $Id: def55bbc9c2a78aef580e88ad6d3f9ba689a6004 $

If you want a real version number you should install a downloaded version.
